# v0.3

## Summary

- Introduz o modelo **app = node (Commoner)** no mobile
- Estabelece fluxo E2E real de capsula (Feira)
- Habilita sincronizacao multi-node via rede (WebSocket)

## Entregas principais

- Commoner executando em ambiente mobile
- Fluxo Feira completo: `offer -> accept -> receipt`
- E2E multi-node com convergencia verificada
- Entrada React Native no SDK JS (sem dependencia obrigatoria de N-API)
- Abstracao inicial de transporte (plugavel)

## Observacoes

- Runtime JS mobile e **nao-canônico** (focado em execucao e DX)
- Runtime Rust/N-API permanece como referencia de validacao
- Trabalho de coerencia/sybil permanece nao-normativo

## Limitacoes conhecidas

- Paridade entre runtimes (JS vs Rust) ainda nao garantida
- Inconsistencias de naming na fronteira TS
- Camada de cliente (AppNode) ainda nao integrada ao SDK

## Impacto

- Permite testes reais em topologia distribuida
- Move o foco de implementacao para comportamento de sistema
- Base concreta para evolucao de DX e transportes

## Proximo foco

- Paridade e conformance entre runtimes
- Robustez de transporte (falhas, latencia, reorder)
- Consolidacao de API (ComumClient)
